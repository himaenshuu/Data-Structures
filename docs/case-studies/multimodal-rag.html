Case study / Multimodal RAG

Unified retrieval and Q&A across documents, web, audio, and video.

This platform supports ingestion, embedding, retrieval, and Gemini-based response generation over multiple content types in one interface.

Role ML systems builder
Content types PDF, URL, papers, audio, video
Engine Gemini + Qdrant RAG

Problem

Most Q&A systems only support one content type, causing fragmented retrieval logic and inconsistent answer quality.

Architecture and stack

  • FastAPI backend for query and retrieval services.
  • Streamlit for rapid, user-facing interaction layer.
  • Qdrant vector store for semantic indexing and retrieval.
  • Gemini for context-aware answer generation.
  • Preprocessing for PDFs, URLs, Grobid-based papers, audio/video transcripts.

Key decisions

  • Used one retrieval abstraction across all ingestion paths.
  • Separated preprocessing from retrieval to keep system modular.
  • Kept vector operations in-memory for rapid experimentation and iteration.
  • Focused on response grounding through context retrieval before generation.

Outcome

  • Created a single Q&A pipeline for five different input formats.
  • Reduced workflow friction for cross-source knowledge extraction.
  • Improved developer velocity by keeping ingestion and retrieval modular.
